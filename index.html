Technical Architecture

RECOMMENDED
TECH STACK

Hybrid orchestration: LangGraph stateful multi-agent + RAG signal retrieval + MCP live data tools. Not pure MCP. Not pure multi-agent. The domain nodes require independent state and memory that only LangGraph provides at production grade.

Orchestration
LangGraph
Stateful graph-based orchestration with checkpointing. Each domain node is a graph node. AICN is a required gate node — no domain node executes without AICN clearance. Supports human-in-the-loop escalation gates.
Why: Only production-grade framework that handles stateful multi-agent workflows with auditable checkpoints at every step.
Reasoning Engine
Claude Sonnet
Primary LLM layer for all domain agent reasoning, synthesis, and executive brief generation. Extended thinking enabled for synthesis layer. Structured output enforced at every node via XML schema.
Why: Best-in-class reasoning with explicit uncertainty acknowledgment. Resists confidence inflation when prompted correctly.
Signal Retrieval
Pinecone + RAG
Vector store for all ingested signals, doctrine documents, think tank reports, and historical intelligence. Hybrid search (dense + sparse) ensures both semantic and keyword retrieval across 15+ signal types.
Why: Persistent signal memory that domain nodes can query independently without cross-contamination.
Signal Ingestion
n8n + MCP
n8n handles scheduled RSS, API polling, and webhook ingestion pipelines. MCP tools provide live web search, maritime AIS data, FRED economic data, and UN trade flow access at runtime.
Why: Separates ingestion infrastructure from reasoning layer. Signals arrive timestamped and bias-tagged before LangGraph sees them.
Structured Storage
PostgreSQL + Redis
PostgreSQL for audit logs, source attribution tables, confidence history, and reversal condition tracking. Redis for real-time signal cache and inter-agent state during active synthesis runs.
Why: Audit defensibility. Every confidence score and source citation is traceable to a specific signal event in PostgreSQL.
Parallel Execution
CrewAI
Manages parallel execution of the 12 domain specialist agents at Layer 3. Each agent runs independently with its own memory context. CrewAI's task delegation ensures no agent contaminates another's output.
Why: LangGraph handles stateful orchestration; CrewAI handles parallel agent delegation. They compose cleanly.
Executive UI
Next.js + Plotly
Next.js serves the executive decision interface. Plotly renders confidence distribution charts, scenario timelines, and escalation risk maps. All outputs carry confidence ceiling labels in the UI — no clean number without a tier badge.
Why: Plotly is the FRED-standard for intelligence visualization. Fast, embeddable, and credible in executive contexts.
API Layer
FastAPI
RESTful API layer connecting LangGraph orchestration to the Next.js UI. Handles authentication, rate limiting, output caching, and brief generation endpoints. Supports PDF export of all executive outputs.
Why: Async-native, fast, and OpenAPI-documented. Every output endpoint is auditable.
Deployment
Docker + Vercel
LangGraph orchestration and CrewAI agents containerized in Docker. Next.js executive UI deployed on Vercel edge. Pinecone and PostgreSQL managed via cloud-native services. Monitoring via Langfuse for agent trace observability.
Why: Langfuse provides trace-level visibility into every agent decision — essential for hallucination auditing.
Anti-Hallucination Architecture

CONFIDENCE
CEILING SYSTEM

No output ships without a confidence tier label. Confidence inflation — presenting LOW-tier conclusions with HIGH-tier certainty — is the primary failure mode this system is designed to prevent.

HIGH
75–100%
Multiple independent corroborating signals. No material contradictions. Track-record-verified sources. Doctrine-anchored. Reversal condition clearly stated.
MODERATE
50–74%
Corroborating signals present but with at least one unresolved contradiction. Source quality varies. Conclusion plausible but not convergent across all domains.
LOW
25–49%
Limited corroboration. Significant contradictions unresolved. High interpretive component. Signal environment sparse for this domain. Use for scenario awareness only.
FRAGMENTED
<25%
Signal environment too contradictory or sparse to support meaningful conclusions. Output is hypothesis only. Domain node flags for human analyst review before synthesis proceeds.
🚫
No Confidence Laundering
Structured output schemas enforce confidence tier labels at every node output. A domain agent cannot return a conclusion without a tier. The synthesis engine cannot aggregate across tiers without downgrading to the lowest contributing tier.
📋
Source Citation Required
Every factual claim in an output must be traced to a signal event in PostgreSQL. The executive UI renders a source count badge on every conclusion. Zero-source claims are blocked at the FastAPI layer.
🔄
Reversal Conditions Mandatory
Every scenario pathway and every domain conclusion must state what evidence would invalidate it. Reversal conditions are displayed in the executive UI alongside the conclusion — not buried in a footnote.
👤
Human Escalation Gates
FRAGMENTED-tier signals trigger mandatory human analyst review before the synthesis engine incorporates them. LangGraph's interrupt() mechanism pauses execution and flags the gate in the UI dashboard.
🔍
Langfuse Trace Observability
Every agent decision, tool call, and confidence assignment is traced in Langfuse. Post-run audits can reconstruct exactly how any conclusion was reached. This is the hallucination forensics layer.
⚖️
Adversary Modeling Protocol
Adversary next-move modeling is constrained to doctrine-anchored behavioral patterns only. No speculative attribution. Every adversary model cites the doctrine source. Outputs are labeled BEHAVIORAL PROJECTION, not INTELLIGENCE ASSESSMENT.
Architecture Decisions

THE 12
DESIGN ANSWERS

Every major architectural question from the system specification, answered with precision.

01 · Multi-agent, MCP, RAG, or hybrid? What orchestration model?
VERDICT: HYBRID — LangGraph stateful multi-agent + RAG + MCP tools.

Pure MCP cannot manage the stateful memory requirements of 12 parallel domain nodes. Pure multi-agent without RAG produces hallucinated signals because agents fabricate what they cannot retrieve. Pure RAG without agents cannot perform the layered reasoning that escalation modeling requires.

The correct architecture:
  • LangGraph as the stateful orchestration backbone — graph nodes for each layer, checkpointing at every gate
  • CrewAI for parallel domain agent execution at Layer 3
  • Pinecone RAG for signal retrieval — agents query, they do not generate signals
  • MCP tools for live data access at ingestion time (web search, maritime AIS, FRED, UN trade)
This is how Palantir's Gotham architecture works at a high level — orchestration layer on top, specialized agents below, retrieval layer as the knowledge base.
02 · Ideal workflow design
SIGNAL → AICN GATE → PARALLEL DOMAIN AGENTS → SYNTHESIS → HUMAN REVIEW GATE → EXECUTIVE OUTPUT

n8n ingestion pipelines run on schedule and webhook triggers. Signals arrive in Pinecone tagged with source metadata. When a query is initiated, LangGraph starts execution: AICN calibration runs first (mandatory gate), then 12 domain agents execute in parallel via CrewAI, then synthesis engine aggregates outputs into scenario pathways. If any domain returns FRAGMENTED confidence, a human review interrupt fires before synthesis proceeds. Final output goes to FastAPI → Next.js executive UI.

Estimated execution time per full synthesis run: 4–8 minutes. Action briefs: 45–90 seconds.
03 · Memory architecture design
Four memory types, each serving a distinct function:

WORKING MEMORY (Redis): Current synthesis run context. Cleared after each run. Prevents cross-run contamination.
EPISODIC MEMORY (PostgreSQL): Historical signal events, past synthesis runs, confidence score history, reversal condition tracking over time.
SEMANTIC MEMORY (Pinecone): All ingested signals, doctrine documents, think tank reports, and policy papers as vector embeddings. Queried by domain agents via RAG.
PROCEDURAL MEMORY (LangGraph graph definition): The WCIS v5.2 and North Star methodologies encoded as graph structure — the system knows how to reason, not just what to retrieve.
04 · How to prevent hallucination and confidence inflation
Six enforcement mechanisms, not one:

  • Structured output schemas at every agent output — confidence tier is a required enum field, not a free-text field
  • Source citation enforcement — FastAPI blocks any output with zero source references from reaching the executive UI
  • AICN mandatory gate — no signal reaches a domain agent without a calibration score; raw signals are never reasoned over directly
  • Confidence downgrade rule — synthesis engine outputs cannot exceed the confidence tier of the lowest-confidence contributing domain node
  • Reversal condition requirement — every conclusion must include a falsification statement or it is rejected by the output schema validator
  • Langfuse trace audit — post-run forensics can identify any hallucinated claim by tracing the agent decision back to its retrieval call
05 · How to modularize WCIS v5.2 and North Star Intelligence
WCIS v5.2 maps directly to the synthesis layer (Layer 4). Its decision superiority framework becomes the scenario pathway generator. Its AICN signal weighting protocol becomes the Layer 2 calibration node. Its adversarial modeling becomes the adversary next-move engine within Layer 4. Its battlespace awareness sub-layer becomes the escalation risk map generator.

North Star Intelligence maps to Layer 1 (signal ingestion) and AICN calibration. Its source registry (10-country tiers, global wire anchors) becomes the source bias metadata table. Its Ad Fontes triangulation protocol becomes a calibration function within AICN. Its narrative polarity mapping becomes a required field on every ingested signal.

Both frameworks are encoded as LangGraph subgraphs — callable as modular components, not hardcoded into the main orchestration flow. This allows version upgrades (WCIS v5.3, North Star v5.0) without rebuilding the architecture.
06 · Executive UI/UX philosophy
Three design rules that make this feel like a defense-adjacent intelligence platform rather than a news dashboard:

  • Every conclusion carries a visible confidence tier badge. No clean number without a tier. The badge is color-coded and always visible — not tucked in a tooltip.
  • Reversal conditions are surface-level, not buried. On every scenario card, the reversal condition appears directly below the conclusion. This forces the reader to engage with uncertainty, not ignore it.
  • Action Briefs are one page maximum. The synthesis engine generates long-form assessments. The executive UI extracts the Action Brief automatically — four sections: Situation, Key Signals, Scenario Pathway, Recommended Watch Items. No scroll required for the brief. Depth is available on demand.
07 · How to structure adversary modeling safely and credibly
Three constraints that keep adversary modeling credible rather than conspiratorial:

  • Doctrine-anchored only. Every adversary behavioral projection must cite a published doctrine, policy document, or historical behavioral pattern. The system cannot project behavior that isn't grounded in a documented precedent.
  • Labeled BEHAVIORAL PROJECTION, not INTELLIGENCE ASSESSMENT. The output schema enforces this distinction. A behavioral projection based on PRC Belt & Road doctrine is not the same as an intelligence assessment of PRC intentions.
  • Confidence ceiling on adversary nodes defaults to MODERATE maximum. Adversary intent is inherently opaque. The system enforces this epistemically — no adversary modeling output can reach HIGH confidence without explicit human analyst override and justification in the audit log.
08 · How this becomes a show-stopping executive platform without appearing extreme
The answer is epistemic discipline, not restraint.

What makes intelligence platforms appear extreme or conspiratorial is when they present low-confidence conclusions with high-confidence framing. HSSCE prevents this architecturally — the confidence ceiling system makes uncertainty visible, which paradoxically increases credibility with sophisticated executive audiences.

The RAND Corporation is trusted not because it is always right but because it is always calibrated. The platform should feel like RAND: conservative in its claims, precise in its uncertainty, and rigorous in its sourcing.

The show-stopping moment is not a dramatic prediction. It is when an executive sees a scenario pathway they hadn't considered, with a clear reversal condition, supported by sources they can verify, at a confidence tier they can trust. That is decision superiority — and no news feed, no geopolitical pundit, and no legacy intelligence dashboard delivers it in that form.
